# CRADLE 2 GRAVE — 近代化・移行計画

## 方針

- **目標:** レガシーHTMLからSSG（静的サイトジェネレーター）ベースに移行し、コンテンツの保守性・可読性・モバイル対応を向上させる
- **UIデザイン:** 全面刷新。旧来のデザイン（テーブルレイアウト・フリー素材画像）は踏襲しない。画像素材（`gazou/`）も再利用しない
- **制約:** 個人運営・低コスト・シンプルさ優先。オーバーエンジニアリングしない
- **前提:** データ系コンテンツが多く、**コンテンツとデータの扱い方の設計が移行の核心**

---

## SSG候補の評価

| SSG | 言語 | データ連携 | 向き・不向き |
|---|---|---|---|
| **Astro** | JS/TS | JSONファイル・APIを`.astro`内で直接扱える。MDX対応 | **最有力。**文章系・データ系の混在に最も柔軟に対応できる |
| **Hugo** | Go | データファイル（JSON/YAML/TOML/CSV）をテンプレートに流し込める | テンプレートの癖が強い。データ系多めなら候補 |
| **11ty (Eleventy)** | JS | データファイルと組み合わせやすい。シンプル | Astroより設定自由度が高い分、初期設定コストが高い |
| **Next.js** | JS/TS | フル機能。SSG/SSR両対応 | このサイト規模にはオーバースペック |

**推奨: Astro**

理由:
- Markdownファイルをそのまま扱える（既存の`.md`資産を活かせる）
- MDXでReactコンポーネントをMarkdown内に埋め込める（データテーブルの動的表示等に有効）
- コンテンツコレクション機能でフォルダ単位のコンテンツ管理が容易
- GitHub Pagesとの相性が良い（GitHub Actionsで自動デプロイ可能）

## データ系コンテンツの扱い方針

レガシーHTMLのテーブルに直書きされているデータをどう扱うかが最大の課題。

### 選択肢

| 方式 | メリット | デメリット | 適用先の目安 |
|---|---|---|---|
| **A. JSONファイルにハードコーディング** | シンプル・外部依存なし | データが多いと管理しにくい | 数十〜百件程度のテーブル |
| **B. CSVファイルで管理** | スプレッドシートで編集可能 | 複雑な構造（ネスト等）が苦手 | フラットな攻略データ |
| **C. ローカルSQLite** | 構造化・クエリ可能 | ビルド時に変換処理が必要 | 非常に大量・複雑なデータ |
| **D. フロント側オブジェクト（JSリテラル）** | ページ内で完結 | SEO・プリレンダリングと相性が悪い | 動的フィルタUI付きページ |

**推奨: A（JSON）+ B（CSV）の組み合わせ**

- フラットなデータ（アイテムリスト等）→ CSV
- 階層・関係性があるデータ（ルート・条件分岐等）→ JSON
- AstroのコンテンツコレクションまたはViteの`import`機能でビルド時に読み込む

### スクレイピングによるデータ抽出

既存HTMLからデータを取り出すための作業：

1. **対象HTMLを特定:** `wildarms/`・`bm/`・`r18/` 等のデータ系ページ
2. **スクレイパーを書く:** Python（`BeautifulSoup`）または Node.js（`cheerio`）で`<table>`要素をパース
3. **出力形式:** CSV or JSONに変換して `src/data/` 以下に配置
4. **確認・補正:** スクレイピング結果を目視確認し、必要に応じて手修正

```python
# スクレイピングのイメージ（Python + BeautifulSoup）
from bs4 import BeautifulSoup
import csv

with open("wildarms/item.htm", encoding="shift_jis") as f:
    soup = BeautifulSoup(f, "html.parser")

rows = []
for tr in soup.select("table tr"):
    cells = [td.get_text(strip=True) for td in tr.find_all("td")]
    if cells:
        rows.append(cells)

with open("src/data/wildarms_items.csv", "w", newline="", encoding="utf-8") as f:
    writer = csv.writer(f)
    writer.writerows(rows)
```

## データ戦略（確定）

- **手動管理・小規模データ → TypeScriptファイル（`src/data/**/*.ts`）**
  - `satisfies` 演算子で型安全を保つ
  - 例: `src/data/ff5/jobs.ts`
- **スクレイパー生成・大量データ → JSONファイル（`src/data/**/*.json`）**
  - Pythonスクリプトが自然にJSONを出力するため
  - 例: wildarms・bm 系
- Firestore等外部NoSQLは**使わない**（SSG＋CDNと相性が悪く、このサイト規模には過剰）
