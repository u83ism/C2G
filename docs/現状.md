## データ戦略

- **手動管理・小規模データ → TypeScriptファイル（`src/data/**/*.ts`）**
  - `satisfies` 演算子で型安全を保つ
  - 例: `src/data/ff5/jobs.ts`
- **スクレイパー生成・大量データ → JSONファイル（`src/data/**/*.json`）**
  - Pythonスクリプトが自然にJSONを出力するため
  - 例: wildarms・bm 系

## ゲームタイトル管理方針

トップメニューとページ内ヘッダーで表記を一致させるため、**ゲームタイトルを単一ソースで管理する**。

### 静的ページ（`.astro`）の場合

`src/data/menu.ts` にエクスポートされた `gameInfo` 定数を単一ソースとする。

```ts
// src/data/menu.ts
export const gameInfo = {
  wildarms: { title: 'ワイルドアームズ', releaseYear: 1996 },
} as const;
```

- `mainSection` の `GameCard.title` / `releaseYear` はここから参照する
- 各ゲームの `.astro` ページも `gameInfo` をインポートして使用する

新しいゲームを追加するときは `gameInfo` にキーを追加し、`mainSection` とページの両方で参照する。

### コレクションベースのページ（Markdown）の場合

Markdownのfrontmatterに `game:` フィールドを定義し、`[slug].astro` から `entry.data.game` として参照する。

```yaml
---
game: ワイルドアームズ
---
```
